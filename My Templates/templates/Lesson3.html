OBJECT ORIENTED PROGRAMMING - OOP

What is a OOP?

OOP - is Object Oriented Programming such as Python, Java and many others. OOP is organized around objects and data rather than actions and logic. It is split into self contained objects. It is almost like having many several mini programs. Each object is represented in a different part of the application, and each object contains it own data and logic. They communicate between themselves. It allows the programmer to think about the program easier. OOP is an idea supported in many languages.

Why is OOP Valuable Tool for Thinking About Large Programming Projects?

OOP is valuable because it allows for programmers to use existing (even other people's) code from one program to another program. This should save time and costs by using existing code where as duplication is limited.

What's the difference between CLASS and OBJECT

CLASS - is the blueprint, the definition, the description it describes what something is, but isn't the thing itself, it is a well defined idea. Classes describes 2 things attributes and behavior. The attributes are the data, and the behavior is the funcitons (what can you do). Also known as properties and methods. It is described in abstract... Classes are self contained units, but represents both data and code that works on the data.

OBJECT - The object is created from the class. Once class can create multiple objects

New Vocabulary Words
  • Instance Variables - are defined inside a method, and they are local to that instance - which is an object all of the variables associated with a specify instance unique to an object... access by using self inside the class, and instance name outside the class
  • Constructor- uses the keyword "self"
  • Self- itself or instance in question
  • Instance Methods- a function that is defined inside a class and associated with an instance
  • Inheritance- is when an object or class is based on another object or class, using the same implementation to maintain the same behavior
  • Method- is a function inside of an object, you can pass attributes (variable inside a method) to the class and method that will get processed inside of the method and independently for each object.
  • Module- is a way to map one thing to another, is a phython file with some functions or variables, you can import that file, and you can access the functions or variables in that modue with the . (dot) operator.
  • Library- is the reference describes the exact syntax and semantics, also can describe some of the optional components that are included.
What is Method Overriding

Allows a child class to provide a specific implementation of a method that is already provided by a parent classes. The implementation in the child class overrides the implementaton in parent class by providing a method that has the same name. class Parent():

def __init__(self, last_name, eye_color):

print("Parent Constructor Called")

self.last_name = last_name

self.eye_color = eye_color

class Child(Parent):

def __init__(self, last_name, eye_color, number_of_toys):

print("Child Constructor Called")

Parent.__init__(self, last_name, eye_color)

self.number_of_toys = number_of_toys
