    Servers

    The purpose of servers is to respond to HTTP requests.
    Client will make a request to the form to the server, a get request.
    The server response with the form data.
    The clent makes a post request to the server, with the data, the post form.
    The server will run the validation functions. It will do one of two things to validate. If the data is good, it will say thanks. If the data or no data is bad, the server will respond with the form data again and with an error message to re-enter the data.

    Validating Input

    Validation is to improve your site's security. Validation verify on the server side is what you expect to receive... servers can receive junk, and you need to deal with it. Validation is an important element for checking what is being entered by the user, you can inform the user of incorrect input by issuing an error message, giving instructions of the problem and how to correct. It can also check for hackers trying to access your system to do harm and corrupt your database/server/system (need to check for bad data inputs). In order to validate the form requires three things:
      1. Verify the user's input
      2. On error, render form again
      3. Include error message
